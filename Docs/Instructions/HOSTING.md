# Hosting

Deployed on **Cloudflare Workers** (free tier, static assets) via Workers
Builds' GitHub integration. Cloudflare deploys on push to `main`; GitHub
Actions runs the CI gate (lint/test/build) and a repository ruleset requires
the `verify` check before merge (repository admins bypass for direct pushes).

**Workers Builds settings** (Cloudflare dashboard → the `skyboxeditor`
Worker → Settings → Build):

| Setting | Value |
| --- | --- |
| Build command | `npm run build` |
| Deploy command | `npx wrangler deploy` (default) |
| Root directory | `/` |

Most configuration is carried in the repo, so the dashboard stays minimal:

- **`wrangler.toml`** declares an assets-only Worker serving `dist/`
  (`[assets] directory = "./dist"`, no `main` entry point, SPA fallback).
- **`.node-version`** pins Node `24` for the build image (kept in sync with
  `.github/workflows/ci.yml`).
- **`public/_headers`** (copied to `dist/_headers`) sets an immutable long
  cache for hashed `/assets/*` and `Access-Control-Allow-Origin: *` plus a
  moderate cache for the cross-origin `/schema/*` JSON schema.
- **`.npmrc`** sets `playwright_skip_browser_download=1`, so the `playwright`
  devDependency does not pull ~300MB of browsers during the Cloudflare build.
  Repo-side — no dashboard environment variable needed. (If a future npm
  drops custom `.npmrc` keys, set `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1` as a
  dashboard build env var instead.)

## Gotchas

- **Lockfile npm version**: GitHub Actions and Cloudflare both run node
  24.18's bundled npm 10.9.2. A `package-lock.json` regenerated by npm 11
  (or bun) can fail `npm ci` there ("Missing: … from lock file"). If it
  happens, regenerate with `npx -y npm@10 install`.
- **Custom domain**: attach skyboxeditor.com to the Worker in the dashboard
  (Settings → Domains & Routes → Custom domain).
